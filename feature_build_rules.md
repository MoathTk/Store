When we start to build new feature you need to do these exact steps:

1. Each file must be clean arch strict.
2. Never hard code any ui , all UIs must be theme aware.
3. Never touch the translations keys file at all, you need to give me the translation keys using (Key:value) format.
4. The feature may contain many things to be added, for the domain and data layer, you build the whole of it instantly,
For the ui screens, widgets and controllers, you need to make it phases, if you need to build 3 screens, with 10  different widgets you divide it to three stages, you build the first screen with only its used widgets , then I review the total job , demand updates or edits if needed then i give you the permission to build the second one , and so on .(to avoid reviewing 2000lines at once which is sometimes highly overwhelming).
5. The screen you build must be only like an orchestrator(I don’t want to see a screen more than 200line of code) to be clean arch adhered.
6. The widgets you create must be considered if they can be reused again .
7. The widgets must be very optimized and looks robust.
8. You will show me a detailed plan for all the process you do and after accepting from me, you will do the same steps here .
9. After you finish building all the feature requirement you will update the projectmap.md.
